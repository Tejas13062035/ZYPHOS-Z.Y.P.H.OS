HOW IT WORKS
1. User types goal via CLI
2. Planner sends goal to Phi-3.5 → gets JSON task list back
3. Executor runs each task using keyword matching → shell/filesystem/sidecar
4. Critic evaluates each result (pass/fail)
5. Memory stores result as vector embedding
6. State saved to disk — resumes after shutdown
